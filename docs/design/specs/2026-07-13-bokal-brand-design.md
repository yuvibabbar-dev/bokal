# Bokal — Brand & Logo Design

**Date:** 2026-07-13 (logo) · 2026-07-14 (renamed Wafer → Bokal) · **Status:** implemented.

## The name

**Bokal** — pronounced *BOH-kal*. It means **"jar / jug"** in several Slavic languages (Serbian,
Croatian, Polish, Slovak). A cookie jar. To an English speaker it reads as a clean coinage; the
meaning is a free gift, not a dependency.

### Why we renamed from "Wafer" (2026-07-14)

Market research (adversarially verified) found "Wafer" was a genuine liability:

- A **funded YC S2025 startup is named exactly "Wafer"** ($4M seed), targets **developers**, and
  sells to the same developer-tooling audience — a direct namespace + SEO collision in our exact market.
- "Wafer" is otherwise crowded in tech (wafer.ee, Waferlabs, Keysight WaferPro, Synopsys), with
  live US software marks WAFERWALLET (Reg. 5952834) and WAFERKEY (Reg. 6580587).
- A solo founder would lose that search race.

Bokal was selected from 75 generated candidates, 39 deep-screened adversarially. It was the
best-positioned survivor on exactly the axis that killed Wafer: **there is zero software
competition for the term.** Runner-up **Kobb** was rejected (spoken aloud it is "Cobb"; and
Kobble.io targets Chrome-extension devs). **Bikkie** was rejected (Bikky Inc. is a $14M-funded
software company with identical phonetics, and "bikkie" literally means "cookie", which poisons
the category's own search terms).

**Verified at rename time (2026-07-14):** `bokal.dev` / `.app` / `.io` all available; npm `bokal`
free; no Chrome Web Store extension named Bokal; no live software trademark found. `bokal.com` is
taken (held since 2002 — skip it; `.dev` is the right apex for a dev tool). `github.com/bokal` is
a personal account → ship the org as `bokal-dev` / `bokalhq`.

### ⚠ Still required before public launch

1. **A real trademark knockout search.** All screening was search-index-derived — USPTO TSDR,
   Justia and Trademarkia all blocked automated access, so "no mark found" means *no evidence
   found*, not *cleared*. Run Class 9 / 42 by hand at `tmsearch.uspto.gov` (BOKAL, BOKALL, BOCAL)
   or pay an attorney $300–800 for a knockout opinion. File **narrowly**: "downloadable browser
   extension software for viewing and managing HTTP cookies."
2. **Lock the namespaces before announcing the name** — `bokal.dev`, npm `bokal`, the GitHub org,
   the Chrome Web Store listing name. Front-running is real: two candidates in the same research
   batch had their domains registered weeks before screening.
3. **Sanity-check the meaning** in Turkish, Hindi, Portuguese, Japanese and Korean before shipping
   to a global store. (Slavic "jar" is benign-to-helpful, but the screening killed two other
   candidates for being obscene in Norwegian and Arabic — so this is not theoretical.)

## The mark

An **amber disc on a graphite rounded-square tile**, with one **flat edge** and chip squares that
read as chocolate chips. Shipped as the extension icon at 16/32/48/128.

The flat edge originally justified the "Wafer" pun (a silicon wafer's orientation flat). **That
rationale retires with the name.** The flat is kept purely because it earns its keep visually: it
gives the disc an asymmetric silhouette, so at 16px the icon doesn't collapse into a generic amber
blob the way a plain circle would.

Two concepts were considered and rejected at design time: **"bitten cookie"** (friendliest, but
visually identical to every other cookie extension) and **"die monogram"** (sharpest at 16px, but
cold — it lost the cookie entirely).

**Open design option (not done):** now that the name means *jar*, the mark could evolve into a
**cookie jar** — which would also hand the Pro tier a natural metaphor (saved cookie profiles =
jars on a shelf). The current disc is a perfectly good cookie and ships fine; the jar is an
upgrade, not a fix.

## Palette

Fixed brand colors — the mark does **not** invert for dark mode (a logo is a constant).

| Token | Hex | Use |
|---|---|---|
| Graphite | `#24282C` | tile background, the flat cutout |
| Graphite (page) | `#1E2226` | landing-page background |
| Graphite (deep) | `#191C1F` | callout / recessed surfaces |
| Amber | `#E9A83E` | the disc; primary accent |
| Amber bright | `#F4C05A` | hover / highlight |
| Chip | `#B67A22` | the chip / chocolate-chip squares |

The extension **UI** accent is still the legacy blue (`--bokal-accent`). Migrating it to amber is a
one-line change in `packages/ui-kit/theme.css` — deliberately deferred so the rename diff stays
reviewable.

## Typography

No custom or licensed font is bundled. The landing page uses the system sans stack. If a display
face is ever wanted for the wordmark, pick one geometric humanist sans and use it *only* for the
wordmark, never for body text.

## Usage & assets

- **Extension icon:** generated by
  [`apps/cookie-manager/scripts/gen-icons.mjs`](../../../apps/cookie-manager/scripts/gen-icons.mjs)
  → `public/icon/{16,32,48,128}.png`. Regenerate with
  `pnpm --filter @bokal/cookie-manager exec node scripts/gen-icons.mjs`.
- **Landing page:** the mark is inlined as SVG in `site/index.html` / `site/privacy.html`, so no
  external asset is required.
- **Minimum clear space:** the graphite tile already provides padding — don't crop it.
- **Do not** recolor the disc, remove the flat edge, or place the mark on a busy background.
