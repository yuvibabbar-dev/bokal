# Wafer — Brand & Logo Design

**Date:** 2026-07-13 · **Status:** implemented (icon shipped), pending founder sign-off on the
open naming question below.

## Decision summary

The logo is **concept A "notched wafer"**: an amber disc on a graphite rounded-square tile with a
single flat edge (a real silicon-wafer orientation flat) and chip/die squares that double as
chocolate chips. It's the only one of the three explored concepts that earns the "Wafer" pun
(silicon wafer + cookie) rather than being a generic cookie, and it keeps a distinctive,
asymmetric silhouette at 16px where a plain disc would just read as a blob.

Two other concepts were considered and rejected: **B "bitten cookie"** (friendliest but visually
identical to every other cookie extension) and **C "die monogram"** (sharpest at 16px but loses the
cookie warmth and reads as pure silicon).

## Palette

Fixed brand colors — the mark does **not** invert for dark mode (a logo is a constant).

| Token | Hex | Use |
|---|---|---|
| Graphite | `#24282C` | tile background, the wafer "flat" cutout |
| Graphite (page) | `#1E2226` | landing-page background |
| Graphite (deep) | `#191C1F` | callout / recessed surfaces |
| Amber | `#E9A83E` | the disc; primary accent |
| Amber bright | `#F4C05A` | hover / highlight |
| Chip | `#B67A22` | the chip/chocolate-chip squares |

The extension **UI** accent remains the existing blue (`--wafer-accent`) for now; migrating the UI
accent to amber is an optional follow-up (a one-line change in `packages/ui-kit/theme.css`), not
done here to avoid churn ahead of the naming decision.

## Typography

No custom/licensed font is bundled. The landing page uses the system sans stack
(`-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, …`). If a display face is ever wanted for
the wordmark, choose one geometric humanist sans and use it *only* for the wordmark, never body.

## Usage & assets

- **Extension icon:** generated by
  [`apps/cookie-manager/scripts/gen-icons.mjs`](../../../apps/cookie-manager/scripts/gen-icons.mjs)
  → `public/icon/{16,32,48,128}.png`. Re-run with
  `pnpm --filter @wafer/cookie-manager exec node scripts/gen-icons.mjs`.
- **Landing page:** the mark is inlined as SVG in `site/index.html` / `site/privacy.html` so no
  external asset is required.
- **Minimum clear space:** the graphite tile already provides padding; don't crop it.
- **Do not** recolor the disc, remove the flat edge, or place the mark on a busy background.

## ⚠ OPEN DECISION — the name "Wafer" (founder call, do not skip)

Market research (session 3, 2026-07-13; adversarially verified) found the name is the **weakest
dimension** and a genuine liability:

- A **well-funded YC S2025 startup is named exactly "Wafer"** ($4M seed; angels incl. Jeff Dean,
  Wojciech Zaremba), targets **developers**, and integrates with Claude Code — a direct namespace +
  SEO collision in Wafer's exact audience (different product category, same search space).
- "Wafer" is otherwise crowded in tech (wafer.ee, Waferlabs, Keysight WaferPro, Synopsys yield
  tools). Live US software trademarks **WAFERWALLET** (Reg. 5952834) and **WAFERKEY** (Reg.
  6580587) exist (the previously-cited "WAFER BROWSER" mark was a misattribution — a cancelled
  furniture mark — so disregard it).
- No fatal trademark blocker was found for a cookie-extension name, **but** organic
  discoverability against a funded, press-covered YC "Wafer" will be an uphill fight for a solo
  founder.

**Recommendation:** decide the name **before** investing in SEO, backlinks, or a public launch.
Options: (a) keep "Wafer" and add a distinctive qualifier + lean on the Chrome Web Store's own
search rather than Google; (b) rename. This logo's amber/graphite system and the disc mark survive
a rename; only the silicon-wafer *pun rationale* is name-coupled. Everything built this session
(icon, README, landing) uses "Wafer" in a small, find-and-replace-able set of places.
